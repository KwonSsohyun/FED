  ▶ 사용자 인터페이스를 만들기 위한 진보된 프레임워크 입니다. 
  
  1. 개념
     - 웹 페이지의 빈번한 변경을 반영하기 위해
       일반적인 JS에서는 요소를 선택 후 하나씩 DOM에 반영한다.

       Vue JS는 메모리상에 있는 "가상 DOM"을 만들어서 변경사항을 모두 수집하여
       실제 DOM에 한번에 반영시켜준다!
       ex) 화장 전체 한 후에 나감 (하나씩 하고 나가는 것이 아닌)

       ※ 가상 DOM이란?
          : 메모리상에만 존재하는 실제 DOM의 카피본이다! (리액트도 마찬가지)

       - 데이터를 실시간으로 변경 반영하도록
         View Model을 사용한다!
    
         ※ 뷰모델(View Model) 이란?
           : 실제 페이지에 반영하는 변경내용을 미리 정의하여
             가상돔에 올리기 위한 모듈이다!


     -> Vue JS의 사용목적
        : 복잡한 웹 어플리케이션(웹)의 개발 및 유지보수를 
          쉽게하기 위해 등장한 JS 라이브러리다!
          + 어플리케이션이라하면 코드를 쳐서 개발한거면 다 어플리케이션이라고 칭함.
            포토샵도 어플리케이션이다.
            웹을 붙혔으니 웹을 뜻함

     -> SPA를 위해서 가장 좋다!
        ※ SPA란? (Single Page Application) 
            : 예컨데 일반 웹페이지에 반복되는 부분과 변경되는 부분이 있다.
            보통 구현시엔 별도의 페이지로 메인페이지, 서브페이지 등으로 만들게 된다!

            그런데 SPA는 한 페이지 안에서 
            고정되는 부분은 그대로 두고,
            변경되는 부분만 Vue JS로 구현할 수 있다!
            (여러장 만들거 한장에 다 만든다.)

  3. Vue JS 기본 구현 구조
     MVVM 모델을 기본으로 함

     [ MVVM ]
       M → "Model" : 화면의 데이터를 구성하는 역할 (어디에다가 뿌릴것인가, 반복할 것인가...)
       V → "View" : 실제 출력화면 (DOM)

       VM → "View Model" : 뷰 인스턴스 객체의 비즈니스 구성하는 역할 (비즈니스=로직)
                           ▶ VM은 모델(M)과 뷰(V)의 중간 역할

     ★★★★★
     정리하자면

     1. ▶ 만들어주는게 "뷰모델(VM)" 
           new Vue({ });  ▶ <script> 구역안에 생성해서 만들고 뿌린다.
        
           이걸 이제 어디에 뿌리는데?
           ▷ 모델구성(가상돔)에 뿌려준다.

    ---------------------------------------------------------------------------------------------

     2. ▶ "Model(M)" 모델구성(가상돔) 을 한다.
           < {{}}  /  v-for  /  v-if  /  v-on:click  /  v-model  /  v-html ... (전용속성명 - 아래 5번에 설명)  ▶ html <body> 구역안에 구성

           이걸이제 Push 한다!
           ▷ 화면에 보이는 (뷰) 에 푸쉬한다!


           ++ {{ }} 콧수염은 text만 넣을 수 있다. (html 태그 처리 불가!) ▶ <h2 v-html="v.타이틀[0]"> 쓴다.

  4. 제이슨(JSON) 은 무엇인가?
     - 데이터를 문자형태로 저장하고, 데이터를 전송하기 위한 파일 포멧이다!

     - 표기법은 JS의 객체 표현법과 동일하다!
       {속성명: 값, 속성명: 값, ....}

     - 파일확장자는 .json

     - 현재 xml 데이터보다 더 많이 사용됨!
       이유는 간단하기 때문
       이건 객체형식으로 적어서 눈에 잘 보이는데
       xml은 꺽세 닫고 열고 하면서 부터 이미 복잡해보임

       ++ F1 눌러서 ->> "기본설정 : 기본설정열기 (JSON)" 을 눌러보면
          VS코드에서도 XML이 아닌 JSON으로 설정파일을 쓰고 있다는 것을 볼 수 있다.
